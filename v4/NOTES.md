# Teaching Notes

## Teaching Method
- **Follow [PROJECT_ARC.md](PROJECT_ARC.md) first, to turn the mission into a lesson sequence.** It decomposes the target artifact into components then concepts, classifies and orders each concept, and sizes lessons using the delete-test and bundling-test from `TEACHING_STYLE.md`. Run before Lesson 1 is written, and revise it whenever teaching reveals it was wrong.
- **Follow [TEACHING_STYLE.md](TEACHING_STYLE.md) for every lesson.** It defines the six-step pipeline (Question the requirement → Ground it in real behavior → See it before symbolizing → Build it from scratch → Compose, never bundle → Compress to plain language) that governs how a concept is taught, regardless of topic.
- **Follow [LESSON-FORMAT.md](LESSON-FORMAT.md) for how that pipeline reaches the page.** Code runs for real in a throwaway scratch surface; every number, table, and plot in the HTML comes from that real run, never invented to look plausible.
- **Consult [LEARNER_ARC.md](LEARNER_ARC.md) across missions, not just within one.** It defines the floor ledger (`~/teach-floor.md`) of transferable patterns already proven, and how scaffolding (never who holds the keyboard) shrinks as that ledger grows. The goal is less pre-explanation needed over time, not any single mission being finished.
- **Ground facts in sources, not memory.** Derive mechanisms from scratch, but gather every external fact, dataset, number, or definition from a real, cited source — never invent from parametric memory.

## User Preferences
- **First-principles learner.** Question the requirement before accepting it — if a "have to" doesn't trace to a fundamental truth, delete it rather than build around it.
- **Physical intuition before symbols.** Every equation should make physical sense first — a picture, a concrete number, a cause-and-effect — before it earns a formula and a name.
- **Mental-model first.** The goal of every lesson is a clear mental model that could be rebuilt from a blank page, not just a working answer.
- **Hands on the keyboard.** Write the naive version and predict outputs before they're revealed — never just watch.
- **Teachability is the success test.** If it can't be explained simply to someone else, the lesson isn't done.
