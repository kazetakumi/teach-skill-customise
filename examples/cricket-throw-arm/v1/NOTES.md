# Teaching Notes

## Teaching Method
- **Follow [TEACHING-METHOD.md](TEACHING-METHOD.md) for every lesson.** It defines the four-phase loop (Visualize → Build → Compress → Retain) and the three gates of understanding. This governs *how* I teach, regardless of topic.
- **Follow [LESSON-FORMAT.md](LESSON-FORMAT.md) for lesson output.** Every lesson ends with an HTML file. Computational topics build in a coupled `.ipynb` first; conceptual topics are HTML only.
- **Check [REVIEW.md](REVIEW.md) at the start of every session.** It is the spaced-review ledger for Phase 4 (Retain). Surface anything due for retrieval *before* teaching new material; update intervals after each review.
- **Style every HTML lesson per [STYLE.md](STYLE.md).** Lessons use the Çengel textbook look (teal banner, objectives box, margin notes, color-boxed key equations). Canonical reference: `_style-preview/cengel.html`.

## User Preferences
- **First-principles learner.** Always derive from the ground up — no accepting results on authority. Start from primitives and build.
- **Mental-model first.** The goal of every lesson is a clear mental model of the concept, not just a working answer.
- **Teachability is the success test.** I should be able to explain the concept to anyone, in simple language, from first principles. If I can't teach it simply, the lesson isn't done.
