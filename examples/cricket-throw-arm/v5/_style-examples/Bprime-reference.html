Lesson 03 · Optimization

Finding the bottom of a bowl

We want the x that makes f(x)=x² smallest. We know it's 0 — but let's make the machine find it, without ever telling it the answer.

The dumbest idea that could work

Stand at some x, feel which way is downhill, step that way. "Downhill" is just the slope — and the slope of x² is 2x, which we write by hand rather than import. Start at x=3 with a step size of 1.0:

def slope(x): return 2*x      # derivative of x², by hand

x, lr = 3.0, 1.0
for i in range(6):
    x = x - lr * slope(x)
    print(round(x, 2))

Predict before you scroll The step is x − 1.0·2x = −x. So from 3 we go to… commit to an answer first.
My guess: 3 → 2 → 1 → 0, slowing near the bottom.

Result3.0 → -3.0 → 3.0 → -3.0 → 3.0 → -3.0
It doesn't descend at all — it bounces between 3 and −3 forever.

The guess was wrong, and the wrongness is the lesson. The step −x doesn't shrink x; it flips its sign. We overshoot the bottom by exactly the amount we started with, every single time. Only one number controls how big that step is — so change that one thing, nothing else, and run it again.

x, lr = 3.0, 0.1          # the one change
for i in range(6):
    x = x - lr * slope(x)
    print(round(x, 2))
Result2.40 → 1.92 → 1.54 → 1.23 → 0.98 → 0.79
Now it slides down the bowl and slows as it nears the bottom.
The same algorithm, same derivative — only the step size changed.

Second angle — same fact, algebraically Substitute the derivative in: x ← x − lr·2x = (1 − 2·lr)x. That's a geometric sequence with ratio (1 − 2·lr). At lr=1.0 the ratio is −1 — it can only flip sign forever. At lr=0.1 the ratio is 0.8 — it shrinks every step. The picture above and this one line of algebra are the same fact, seen two ways.

The only thing between "bounces forever" and "finds the minimum" was the size of the step. That single knob is the learning rate.

And notice it slows down on its own: as x shrinks, the slope 2x shrinks with it, so the steps shorten near the answer without any extra rule. The descent self-brakes.

Could you rebuild it?

On a blank page, with no notes: why did lr=1.0 bounce but lr=0.1 descend?

And what's the largest lr that still converges? Don't reason it out — change one thing and look.
